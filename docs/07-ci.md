# CI

All workflow files live in `.github/workflows/`.
Every workflow targets a specific class of change, which keeps CI fast: a change to the kernel does not trigger the tools test run, and a change to the tools does not trigger the Doxygen build unless the relevant paths have been modified, as well.

## build.yaml — Kernel Build Matrix

Triggers on every pull request targeting `main`.
The matrix builds the kernel across all four supported preset configurations to catch any combination of architecture and compiler that might introduce a regression:

| Preset          | Architecture | Compiler |
|-----------------|--------------|----------|
| `x86_64-gcc`    | x86_64       | GCC      |
| `x86_64-clang`  | x86_64       | Clang    |
| `aarch64-gcc`   | aarch64      | GCC      |
| `aarch64-clang` | aarch64      | Clang    |

The `x86_64-clang` cell carries two additional quality gates.
It runs `clang-format --dry-run --Werror` over every `kernel/**/*.{c,h}` file; any file that diverges from the `.clang-format` configuration fails the check immediately.
It also runs clang-tidy against the generated `compile_commands.json`.
Any finding that clang tidy classifies as an error aborts the job.

## tools.yaml — Tools Build, Test, and Coverage

Triggers on pull requests and on pushes to `main` when any file under `tools/` changes.
It runs in two parallel jobs.

The **build-and-test** job builds the tools sub-project with both GCC and Clang and then runs the full CTest suite with `--output-on-failure`.
Both compilers must produce a clean build and all tests must pass.

The **coverage** job builds the tools with GCC and gcov instrumentation, runs all tests, and produces an HTML coverage report.

## doxygen.yaml — API Documentation

Triggers on pull requests and on pushes to `main` when `kernel/**`, `tools/**`, or `Doxyfile` changes.

The **build** job generates Doxygen HTML and uploads the `doxygen-html` artifact (30-day retention).
It runs on every push and every pull request, so any documentation regression is caught before merge.

The **publish** job runs only on pushes to `main`.
It deploys the Doxygen HTML directly to GitHub Pages at `/JANUS/`, making the API reference available to all contributors without any manual deployment step.
